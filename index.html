  2. Non-preemptive scheduling

üîí Deadlock Management

Deadlock Conditions (Coffman Conditions)

Four Necessary Conditions:

  1. Mutual Exclusion: Resources cannot be shared
  2. Hold and Wait: Process holds resources while waiting
  3. No Preemption: Resources cannot be forcibly taken
  4. Circular Wait: Circular chain of waiting processes
Deadlock Example

‚Üí
Resource R1
R1 held by P1
R2 held by P2

Circular Wait: P1 waits for R2 (held by P2), P2 waits for R1 (held by P1)

Deadlock Handling Strategies

Prevention:

  1. Eliminate one of the four conditions
  2. Resource ordering to prevent circular wait
  3. Banker's algorithm for safe allocation

Detection & Recovery:

  1. Use resource allocation graph
  2. Process termination
  3. Resource preemption
Banker's Algorithm

Safe State Algorithm:

  1. Check if system is in safe state
  2. Find process that can complete with available resources
  3. Assume it completes and releases resources
  4. Repeat until all processes complete

Components:

  1. Boot Block: Contains boot loader
  2. Super Block: File system metadata
  3. Inode Table: File metadata and pointers
  4. Data Blocks: Actual file content
File Allocation Methods

Contiguous Allocation:

  1. Files stored in consecutive blocks
  2. Fast access but external fragmentation

Linked Allocation:

  1. Files stored as linked list of blocks
  2. No external fragmentation but slow random access

Indexed Allocation:

  1. Index block contains pointers to data blocks
  2. Fast random access, supports large files
